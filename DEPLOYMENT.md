# 🚀 Job Portal Deployment Guide (Free Tier)\n\nThis guide will help you deploy your job posting portal using the free tier of Vercel (frontend) and Railway (backend).\n\n## 📋 Pre-Deployment Checklist\n\n- [ ] Code pushed to GitHub repository\n- [ ] Environment variables configured\n- [ ] Database schema ready\n- [ ] Build scripts tested locally\n\n## 🛤️ Backend Deployment (Railway)\n\n### Step 1: Sign up for Railway\n1. Go to [railway.app](https://railway.app)\n2. Sign up with GitHub\n3. Connect your repository\n\n### Step 2: Deploy Backend\n1. Click \"New Project\" → \"Deploy from GitHub repo\"\n2. Select your repository\n3. Choose \"Deploy from subfolder\" → Select `backend`\n4. Railway will auto-detect Node.js and PostgreSQL\n\n### Step 3: Configure Environment Variables\nIn Railway dashboard → Variables:\n```env\nNODE_ENV=production\nJWT_SECRET=your-super-secret-jwt-key-minimum-32-characters\nJWT_EXPIRES_IN=7d\nJWT_REFRESH_EXPIRES_IN=30d\nCORS_ORIGIN=https://your-app-name.vercel.app\nPORT=3001\n```\n\n### Step 4: Database Setup\n1. Railway auto-provisions PostgreSQL\n2. Get DATABASE_URL from Railway dashboard\n3. Run migrations:\n   ```bash\n   # In Railway console or locally with production DB\n   npm run migrate\n   npm run seed\n   ```\n\n## 🌐 Frontend Deployment (Vercel)\n\n### Step 1: Sign up for Vercel\n1. Go to [vercel.com](https://vercel.com)\n2. Sign up with GitHub\n3. Install Vercel CLI: `npm i -g vercel`\n\n### Step 2: Deploy Frontend\n1. Go to Vercel dashboard\n2. Click \"New Project\"\n3. Select your GitHub repository\n4. Set these configurations:\n   - **Root Directory**: `frontend`\n   - **Build Command**: `npm run build`\n   - **Output Directory**: `dist`\n   - **Install Command**: `npm install`\n\n### Step 3: Configure Environment Variables\nIn Vercel dashboard → Settings → Environment Variables:\n```env\nVITE_API_URL=https://your-backend-name.up.railway.app/api\n```\n\n### Step 4: Custom Domain (Optional)\n1. In Vercel dashboard → Domains\n2. Add your custom domain\n3. Update DNS records as instructed\n4. Update CORS_ORIGIN in Railway to match new domain\n\n## 🔧 Post-Deployment Steps\n\n### 1. Test Your Deployment\n- [ ] Visit your frontend URL\n- [ ] Test user registration\n- [ ] Test login functionality\n- [ ] Test job posting/searching\n- [ ] Check mobile responsiveness\n\n### 2. Set Up Monitoring\n```bash\n# Add basic error tracking\nnpm install @sentry/react @sentry/node\n```\n\n### 3. Configure Production Settings\n- [ ] Set up SSL (automatic with Vercel/Railway)\n- [ ] Configure CORS properly\n- [ ] Set up error logging\n- [ ] Add analytics (Vercel Analytics is free)\n\n## 🎯 Free Tier Limits\n\n**Vercel Free:**\n- 100GB bandwidth/month\n- 6,000 build minutes/month\n- Custom domains included\n\n**Railway Free:**\n- $5 credit/month\n- Covers small PostgreSQL database\n- Auto-sleep after inactivity\n\n## 🚨 Production Optimizations\n\n### Backend (Railway)\n1. **Keep Services Awake**:\n   ```bash\n   # Add to package.json\n   \"scripts\": {\n     \"start\": \"node dist/index.js\",\n     \"keepalive\": \"node keepalive.js\"\n   }\n   ```\n\n2. **Create keepalive.js**:\n   ```javascript\n   // Prevents Railway from sleeping\n   setInterval(() => {\n     fetch(process.env.RAILWAY_PUBLIC_DOMAIN + '/health')\n   }, 14 * 60 * 1000); // Every 14 minutes\n   ```\n\n### Frontend (Vercel)\n1. **Enable Analytics**:\n   - Go to Vercel dashboard → Analytics\n   - Enable Web Analytics (free)\n\n2. **Optimize Performance**:\n   ```javascript\n   // Add to vite.config.ts\n   export default defineConfig({\n     build: {\n       rollupOptions: {\n         output: {\n           manualChunks: {\n             vendor: ['react', 'react-dom'],\n             router: ['react-router-dom']\n           }\n         }\n       }\n     }\n   })\n   ```\n\n## 🎉 Go Live Checklist\n\n- [ ] Backend deployed and responding\n- [ ] Frontend deployed and accessible\n- [ ] Database connected and seeded\n- [ ] Environment variables set\n- [ ] CORS configured correctly\n- [ ] SSL certificates active\n- [ ] Basic monitoring enabled\n- [ ] Error handling working\n- [ ] Mobile responsive\n- [ ] Cross-browser tested\n\n## 🆘 Common Issues\n\n**CORS Errors:**\n- Check CORS_ORIGIN matches your frontend domain exactly\n- Include protocol (https://)\n\n**Database Connection:**\n- Verify DATABASE_URL is set in Railway\n- Check if migrations ran successfully\n\n**Build Failures:**\n- Ensure all dependencies in package.json\n- Check TypeScript errors\n- Verify build scripts work locally\n\n**Railway Sleeping:**\n- Free tier apps sleep after 30min inactivity\n- First request after sleep takes ~30 seconds\n- Consider keep-alive scripts or upgrade to paid tier\n\n---\n\n🎊 **Congratulations!** Your job portal is now live and accessible to users worldwide!\n\n**Your URLs:**\n- Frontend: `https://your-app-name.vercel.app`\n- Backend: `https://your-backend-name.up.railway.app`\n- Database: Managed by Railway\n\n**Next Steps:** Consider adding features like email notifications, file uploads, or premium job postings as your user base grows!
